Light Approaches for High-Efficiency Python

Posted on 2017-10-09 | Views   |    Views

Python is known for easy coding and calling c/c++ libraries. However, the first point is at the expense of executing speed. You also need to strictly follow Python instructions to call c/c++ libraries.

So, here comes a question: Can we make Python run much faster? The answer is definitely YES!

Before introduce the light approaches to speed up Python, I want to show a little examples – sum up a 2-dimension array.

Original Python:

import numpy as np

def sum2d(arr):
       n, m = arr.shape
    ret = 0.0
    for i in xrange(n):
        for j in xrange(m):
            ret += arr[i, j]
    return ret

Python with Numba:

from numba import jit
import numpy as np

@jit
def sum2d(arr):
    n, m = arr.shape
    ret = 0.0
    for i in xrange(n):
        for j in xrange(m):
            ret += arr[i, j]
    return ret

Cython:

import numpy as np
cimport numpy as np
import cython

@cython.boundscheck(False)
@cython.wraparound(False)
cdef double _sum2d(np.ndarray[np.float32_t, ndim=2] arr):
    cdef int n, m
    n = arr.shape[0]
    m = arr.shape[1]
    cdef double ret = 0.0
    for i in range(n):
        for j in range(m):
            ret += arr[i, j]
    return ret

def sum2d(arr):
    return _sum2d(arr)

The three code snippets seem to be quite similar, but let’s see the running time.

Numba and Cython’s running time are almost the same and it’s about 290 times faster than original Python. You may not believe me if I say that it runs even faster than trivial C++ code. Let’s do it!

Trivial C++:

#include <ctime>
#include <cstdlib>
#include <chrono>
#include <iostream>

class Matrix {
    float *data;
public:
    size_t n, m;
    Matrix(size_t r, size_t c): data(new float[r*c]), n(r), m(c) {}
    ~Matrix() { delete[] data; }
    float& operator() (size_t x, size_t y) { return data[x*m+y]; }
    float operator() (size_t x, size_t y) const { return data[x*m+y]; }
};

float sum2d(const Matrix &a) {
    float ret = 0;
    for (size_t i = 0; i < a.n; ++i) 
        for (size_t j = 0; j < a.m; ++j) {
            ret += a(i, j);
        }
    return ret;
}

void fill(Matrix &a) {
    int cnt = -1;
    for (size_t i = 0; i < a.n; ++i)
        for (size_t j = 0; j < a.m; ++j) {
            a(i, j) = ++cnt;
        }
}

int main() {
    srand((unsigned)time(NULL));
    const int n = 1000, m = 100000, T = 10;
    Matrix x(n, m);
    fill(x);
    auto st = std::chrono::system_clock::now();
    float s = 0;
    for (int i = 0; i < T; ++i) {
        s += sum2d(x);
    }
    auto ed = std::chrono::system_clock::now();
    std::chrono::duration<double> diff = ed-st;
    std::cerr << s << std::endl;
    std::cout << T << " loops. average " << diff.count() * 1e3 / T << "ms" << std::endl;
}

The results are amazing:

You see, C++ is even slower. The reason may be that the excellent optimization of Numpy!

Through the example, we could believe that there are many Light-Approaches for speeding up Python code. I will introduce two popular tools in the next sections, Numba and Cython.

Numba

Numba utilizes just-in-time(JIT) compilation, which could finish compilation during execution. So numba could infer the type of your Python object(variable) during your execution. People may ask here, why mainstream Python compiler do not support this feature? There are too many history problems here. Nowadays, the most popular python Interpreter is CPython who uses a GIL(GlobalInterpreterLock). Some minor Interpreters are trying to do these things, but there are still catching up, e.g PyPy.
